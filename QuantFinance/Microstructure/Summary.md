# 订单流微观分类总结：从原始数据到 Alpha 因子

> 本文是 [Microstructure 模块](README.md) 的总结篇，将六个独立专题串联成一条完整的逻辑主线。

---

## 一、问题的本质

现代金融市场不再是一个抽象的宏观定价机制，而是被解构为**由海量异质性订单构成的复杂动力学系统**。

数据从日线/分钟线下沉到 **Tick / MBO（Market-by-Order）** 级别后，量化研究的核心问题从"价格涨跌"转变为：

> **每一笔订单背后是谁？他/它在想什么？这股流量将把市场推向何方？**

这就是订单流微观分类的核心问题。

---

## 二、五个分类维度的全景图

订单流的分类可以从五个独立但相互关联的维度展开。每个维度回答一个不同的问题：

```
                    一笔成交事件
                          │
        ┌─────────────────┼─────────────────┐
        │                 │                 │
        ▼                 ▼                 ▼
   ① 谁发起的？      ② 谁在提供流动性？  ③ 这股流有信息吗？
  Lee-Ready/BVC      Maker/Taker        PIN/VPIN
                     Kyle 模型           毒性度量
        │                 │                 │
        └─────────────────┼─────────────────┘
                          │
              ┌───────────┴───────────┐
              │                       │
              ▼                       ▼
        ④ 谁是发起者？           ⑤ 盘口压迫力如何？
       Iceberg / Sweep         OFI / VOI
       Boehmer 散户识别         (动态/静态)
```

## 三、五维度的递进逻辑

### 维度一：交易方向 — 还原最基本的"主动方"

> **没有方向，就没有失衡；没有失衡，就没有 Alpha。**

| 算法 | 适用场景 | 哲学 |
|------|---------|------|
| **Lee-Ready** | 学术研究，高精度因子验证 | 二元离散判定（绝对买/卖） |
| **EMO** | 连续同区间多笔交易 | Lee-Ready 的边界优化 |
| **BVC** | 实时高频，仅有 K 线数据 | 概率连续切分（按比例分配） |

→ 详见 [Lee-Ready & BVC](TradeDirection_LeeReady_BVC.md)

### 维度二：流动性力学 — 谁在喂养市场？

把所有参与者映射为**流动性提供者 (Maker)** 和**流动性消耗者 (Taker)** 两种物理角色。

**Kyle (1985) 模型**给出了这场博弈的严格数学描述：

$$\lambda = \frac{1}{2}\sqrt{\frac{\Sigma_0}{\sigma_u^2}}$$

| 经济直觉 | 数学含义 |
|---------|---------|
| 资产波动越大 → 流动性越差 | $\lambda \propto \sqrt{\Sigma_0}$ |
| 噪音交易越多 → 流动性越好 | $\lambda \propto 1/\sqrt{\sigma_u^2}$ |

**核心洞察**：任何 Taker 在消耗流动性时，都在向 Maker 支付**信息风险溢价**。

→ 详见 [Kyle 模型推导](KyleModel_Derivation.md)

### 维度三：信息毒性 — 这股流是否有"知情者"？

做市商最怕的不是波动，而是**有毒订单流 (Toxic Order Flow)** —— 单方向、有持续性、有信息。

**演化路径**：

```
PIN (1996) ──→ VPIN (2012)
  │              │
  时间时钟      体积时钟
  MLE 估计      解析计算
  日级别        毫秒级
```

**VPIN 的封神案例**：2010 年闪电崩盘前一个多小时，VPIN 已突破 0.90 的极度危险阈值，提前发出了红色警报。

→ 详见 [PIN / VPIN](PIN_VPIN.md)

### 维度四：参与者画像 — 谁是订单背后的人？

这一维度通过**反向工程**给匿名订单打上行为标签：

| 类型 | 行为 | 识别方法 | Alpha 价值 |
|------|------|---------|-----------|
| **冰山订单** | 隐蔽、缓慢、持续 | MBO 追踪 + 时间差 + 生存分析 | 跟随机构方向 |
| **扫单流 (ISO)** | 狂暴、瞬时、推土机 | 单笔大单穿透多档位 | 动量确认信号 |
| **散户订单** | 微小、分散、群体效应 | Boehmer 子便士算法 | 正向 Alpha (~10 bps/周) |

**最颠覆性的发现**：Boehmer 等人证明，**散户净买入不是反向指标，而是正向 Alpha 信号**。这彻底颠覆了"散户是无害噪音"的传统认知。

→ 详见 [Iceberg & Sweep](OrderTypes_Iceberg_Sweep.md) | [Boehmer 散户识别](RetailOrders_Boehmer.md)

### 维度五：盘口失衡 — 终极 Alpha 因子

经过前四个维度的还原，最终我们得到了真正可以喂给执行算法和高频自营策略的 Alpha 因子：

| 因子 | 类型 | 视角 | 特性 |
|------|------|------|------|
| **OFI** | 动态流量 (Flow) | 一段时间内的事件积分 | 强线性预测 $\Delta P$ |
| **VOI** | 静态存量 (Stock) | 时点快照 | 简单清晰，适合实时风控 |

**OFI vs 简单"交易方向"的本质区别**：
- 交易方向只看**已成交的部分**
- OFI 还看**撤单流和挂单流**——这是真正具有前瞻性的信息

**$\Delta P = \beta \cdot \text{OFI} + \epsilon$** —— 一个堪比物理定律的强线性关系。

**前沿发展**：将 LOB 多档数据组成张量矩阵，喂给 **DeepLOB** (CNN + LSTM) 等深度学习架构，捕捉传统线性回归无法触达的非线性时空特征。

→ 详见 [OFI / VOI](OFI_VOI.md)

## 四、底层数据的依赖关系

不同维度对数据颗粒度的需求差异巨大：

| 维度 | 最低数据要求 | 理想数据要求 |
|------|-------------|-------------|
| Lee-Ready | L1 + 成交 | L1 + 成交 + 纳秒时间戳 |
| BVC | K 线 (Bar Data) | K 线 |
| Kyle / VPIN | L1 + 成交 | L1 + 成交 |
| 冰山检测 | L2 | **MBO** (每个 Order ID) |
| 散户识别 (Boehmer) | TAQ + 子便士成交价 | (仅美国市场) |
| OFI | L1 + 撤单/挂单事件 | L2 + MBO |
| VOI | L1 快照 | L1+ 多档快照 |
| DeepLOB | L2 (10+ 档) | L2 (20+ 档) + 高频 |

**数据预算与因子能力成正比**——这就是为什么顶尖 HFT 公司愿意为 MBO 数据每月支付数十万美元：能识别出冰山订单的一次正确判断，可能就是数百万美元的 Alpha。

## 五、各模块的相互连接

```
       BVC ─────────┐
        │           │
        │           ▼
   交易方向 ────→ VPIN ────→ 闪电崩盘预警
        │           ▲
        │           │
        │      做市商风控
        │           │
        ▼           │
       OFI ←────────┤
        │           │
        │      Kyle 模型
        │      (理论基础)
        │           │
        ▼           ▼
   预测 ΔP    最优执行策略
                    ▲
                    │
                   VOI
```

- **BVC** 是 VPIN 的方向分类底层
- **Kyle 模型**为 VPIN 的"毒性"概念提供理论基础
- **OFI** 利用了交易方向 + 撤单/挂单流
- **VOI** 是 LOB 静态版本的失衡指标，与 OFI 互补
- **冰山/扫单/散户识别**是参与者画像，与失衡指标交叉使用产出更精细的因子

## 六、给量化研究员的实战建议

### 起步阶段
1. 先掌握 **Lee-Ready** 的判定逻辑，理解"主动方"的意义
2. 用日级别数据复现 **PIN** 模型，体会 MLE 的痛苦
3. 在分钟 K 线上实现 **BVC + VPIN**，绘制毒性曲线

### 进阶阶段
4. 在 L1 数据上实现 **OFI**，回归 $\Delta P$，看 $R^2$
5. 在 L2 数据上实现 **VOI**，与 OFI 对比预测能力
6. 用 MBO 数据尝试**冰山检测**

### 前沿阶段
7. 多档 LOB → DeepLOB 深度学习
8. 跨资产 / 跨市场的 OFI 联合建模
9. 结合 [因子评价指标](../Basics/IC_IR_Sharpe.md) 完整测试因子的 IC、IR 与稳定性
