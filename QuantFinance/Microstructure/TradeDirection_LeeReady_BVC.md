# 交易方向推定：Lee-Ready 与 BVC 算法

## 为什么需要推定交易方向？

在金融数据分析的基础阶段，尤其是面对**缺乏直接交易方向标记的历史 L1 或早期 L2 数据**时，推断某一笔已执行的成交是"主动买入"还是"主动卖出"，**是构建所有后续微观结构因子的先决条件**。

这一分类体系旨在**还原交易的原始发起端**，进而计算出真实的市场订单流失衡度。没有方向，就没有 OFI；没有 OFI，就没有现代高频 Alpha 工厂的所有上层建筑。

> **直觉**：成交本身没有方向。是 A 主动找 B 成交，还是 B 主动找 A 成交？同一笔成交事件，在两种解释下产生的微观信号是完全相反的。我们必须找到一种方法，从价格、时间、报价等"周边线索"反推出真实的发起方。

---

## 一、Lee-Ready 算法 (1991) — 黄金标准

在学术界与工业界，**Lee-Ready (1991) 算法**长期以来占据着推断交易方向的**黄金标准**地位。

该算法的精妙之处在于：它不仅依赖于成交价格的绝对变动，还**结合了成交瞬间的微观报价环境**。通过两个层次的规则——**报价规则 (Quote Rule)** 和**剔除规则 (Tick Test)**——构建了一套严密的决策树逻辑：先尝试用更精确的报价信息，失效后再回退到价格趋势。

### 1.1 符号定义

- $P_t$：$t$ 时刻的实际成交价格
- $P_t^A, P_t^B$：交易瞬间的最佳卖价 (Ask) 与最佳买价 (Bid)
- $M_t = \dfrac{P_t^A + P_t^B}{2}$：买卖中间价 (Mid-price)

### 1.2 报价测试 (Quote Rule)

报价测试的核心思想是：**成交价相对于中间价的偏离方向，反映了发起方为了"立即成交"愿意付出的代价**。

将成交价与中间价比较：

| 条件 | 判定 | 物理直觉 |
|------|------|---------|
| $P_t > M_t$ | **主动买入** (Buyer-Initiated / Ask-taker) | 发起方愿意支付**高于市场平均水平**的溢价来获取流动性 → 必然是急于成交的买方，即"吃掉"卖单 |
| $P_t < M_t$ | **主动卖出** (Seller-Initiated / Bid-taker) | 发起方愿意以**低于市场平均水平**的折价立即出货 → 必然是急于成交的卖方，即"吃掉"买单 |
| $P_t = M_t$ | 进入剔除测试 | 报价规则失效，价格刚好处于中间位置无法判断 |

> **本质**：流动性的消耗是有成本的，谁付了这个成本，谁就是发起方。

### 1.3 剔除测试 (Tick Test)

当成交价正好位于中间价（即 $P_t = M_t$），或者市场缺乏实时准确的报价数据时，**报价测试便会失效**。此时算法自动回退到剔除测试机制。

剔除测试通过比较当前成交价 $P_t$ 与上一次**不同价格**的成交价 $P_{t-1}$ 的相对关系来推定方向：

| 条件 | 名称 | 判定 | 直觉 |
|------|------|------|------|
| $P_t > P_{t-1}$ | **Uptick**（向上推移） | 主动买入 | 价格被推上去 → 必有买方付出代价 |
| $P_t < P_{t-1}$ | **Downtick**（向下推移） | 主动卖出 | 价格被压下去 → 必有卖方付出代价 |

> **设计哲学**：Quote Rule 是更精确的"现场观察"，Tick Test 是更稳健的"轨迹回溯"。当现场证据不足，就转用历史轨迹推断。

### 1.4 实证准确率

实证研究表明，这套复合算法在**不同市场环境下均表现出较高的鲁棒性**。例如，针对德国股票市场（其交易机制具有代表性）的一项深度测试显示：

- **Lee-Ready 算法能够正确分类约 72.8% 的交易记录**
- 单独使用简单的 Tick Test 也能达到近似的准确率

> 这里有一个值得思考的发现：单用 Tick Test 已经接近完整算法的准确率。这说明价格变动的方向本身就是非常强的信号，Quote Rule 的边际增益主要在于消除特定边界情况的偏差。

### 1.5 EMO 算法 (Ellis-Michaely-O'Hara)

除了经典的 Lee-Ready 算法外，**Ellis-Michaely-O'Hara (EMO) 算法**进一步对该方法进行了优化，提供了更为稳健的分类规则。

EMO 的关键改进点在于：当多笔交易**连续发生在同一买卖报价区间内**时，原始 Lee-Ready 算法在处理这种"被同一波订单流连续吃掉"的场景下会产生分类偏差。EMO 通过更精细的边界条件处理，在这类高度连续的交易序列中表现更为出色。

### 1.6 高频时代的微观挑战

尽管 Lee-Ready 算法在理论上极具解释力，但在现代高频交易环境中，它面临着**严峻的微观挑战**：

**挑战 1：时间戳错位**

在碎片化的高频市场中，交易所系统的时间戳精度若不足以完美对齐**毫秒或微秒级**的报价（Quote）与成交（Trade）数据流，**极微小的时间错位**就会导致 $M_t$ 的选取错误。

举个例子：如果在 09:30:01.123 ms 发生了一笔成交，但用于比对的 Quote 实际上是 09:30:01.119 ms 的旧报价（中间已有更新），那么算出的 $M_t$ 就是过时的，整个判断就会出现系统性偏差。这种偏差在统计上可能放大到数个百分点。

**挑战 2：计算开销**

对于每秒产生**数万笔订单记录**的现代交易引擎而言，逐笔回溯与比对的计算开销极为庞大。Lee-Ready 是一个状态机算法，需要持续追踪：

- 当前最优买卖报价的实时更新
- 上一次"不同价格"的成交记录
- 时间戳的精确对齐

这极大地限制了它在**实时高频风控**中的应用。学术分析（事后处理几个月数据）尚可，但要在毫秒级风控系统中实时跑这套算法成本极高。

| 挑战 | 后果 |
|------|------|
| 时间戳精度不足以对齐毫秒/微秒级 Quote 与 Trade | $M_t$ 选取错误，引发系统性分类偏差 |
| 每秒数万笔订单的逐笔回溯计算 | 计算开销庞大，不适合实时高频风控 |

## 二、BVC — 批量订单分类算法

### 2.0 范式革命

为了应对高频交易时代的庞大数据量与微观噪音，**Easley、López de Prado 和 O'Hara (ELO)** 提出了一种**范式转换级别的解决方案**：批量订单分类法（**Bulk Volume Classification**, BVC）。

BVC 的革命性在于它**摒弃了对每一笔离散交易进行绝对二元定性（0 或 1）的传统思路**，转而采用**概率统计学的框架**：

> 把一段时间内（如 1 分钟）或指定交易量区间内的**总成交量**，按**比例近似切分**为主动买入量和主动卖出量。

这是一个根本性的视角转变：
- **Lee-Ready**：每一笔订单非黑即白，要么 100% 是主动买，要么 100% 是主动卖
- **BVC**：一个区间内必然混合了买单和卖单，我们用概率分布做柔性切分

### 2.1 核心公式

第 $\tau$ 个聚合区间 (Bucket) 内，被分类为主动买入的连续交易量：

$$V^B_{\tau} = \sum_{i = \text{last}(\tau-1)+1}^{\text{last}(\tau)} V_i \cdot Z\left(\frac{\Delta P_i}{\sigma_{\Delta P}}\right)$$

| 符号 | 含义 |
|------|------|
| $\tau$ | 当前批量区间索引 |
| $i$ | 桶内子时间柱 / 数量柱 (Bar) 的迭代索引 |
| $V_i$ | 第 $i$ 个 Bar 的总物理交易量 |
| $\Delta P_i$ | 该子区间内的标准化价格变动 |
| $\sigma_{\Delta P}$ | 价格变动的样本标准差（短期波动率基准） |
| $Z(\cdot)$ | 累积分布函数 (CDF)，正态或 t 分布 |

### 2.2 为什么用 CDF？

在这里使用累积分布函数（CDF，即公式中的 $Z(\cdot)$），核心目的是将分式计算出的"标准化的价格变动"**平滑地映射为一个介于 0 到 1 之间的概率值**（或百分比）。

使用正态分布或 t 分布的累积分布函数，**完美契合了订单流失衡的深层微观物理直觉**：

#### 三种状态的物理直觉

| 价格变动 | $Z(\cdot)$ 值 | 切分结果 | 直觉 |
|----------|--------------|----------|------|
| $\Delta P_i = 0$（绝对均衡） | $Z(0) = 0.5$ | 50% 买 / 50% 卖 | 价格没动，买卖势均力敌 |
| $\frac{\Delta P_i}{\sigma_{\Delta P}} \gg 0$（单边拉升） | 接近 1 | ~100% 买入 | 暴涨必由激进市价买盘驱动 |
| $\frac{\Delta P_i}{\sigma_{\Delta P}} \ll 0$（单边砸盘） | 接近 0 | ~0% 买入 → 全部卖出 | 暴跌必由市价卖盘驱动 |

CDF 提供了一条连续的 S 型曲线，**不纠结于逐笔数据的离散判定**，而是以**宏观统计的视角**，优雅地将"价格异动的激烈程度"直接转化为"主动订单流的倾斜比例"。

> **数学直觉**：S 型曲线的中间区域是线性变化的（轻度失衡时按比例分配），而两端是渐进饱和的（极端行情时几乎全部归为一方）。这恰好符合金融市场的真实行为：温和波动时买卖力量混合，极端行情时单方力量主导。

### 2.3 BVC 的工程优势

BVC 的这种设计**极大地降低了数据处理的复杂度和时间延迟**：

- **不需要逐笔报价数据**：使得量化策略在仅具备 K 线数据 (Bar Data) 而缺乏 Level-1 Tick 数据的情况下，依然能够高效地近似重构出高频订单流失衡
- **不需要纳秒级时间戳对齐**：避免了 Lee-Ready 算法的时间错位问题
- **天然平滑微观噪音**：因为是按比例切分而非二元判断，单笔异常订单的影响被自然稀释
- **专为现代高频架构设计**：可以在毫秒级风控系统中实时运行

> **实战意义**：很多中等规模的量化机构买不起最贵的 Tick 级数据源，但能够获取免费或便宜的分钟 K 线。BVC 让这些机构也能近似重构出订单流失衡因子，是 ELO 团队对量化民主化的重要贡献。

### 2.4 学术争议：速度 vs 精度的权衡

学术界对 BVC 的绝对优越性并未达成一致。部分实证研究指出：

- **BVC 处理速度极快**，工程友好
- 但在股票市场的微观准确度 (Accuracy Rate) 测试中，**经典 Lee-Ready 和 Tick Rule 仍能产生更优质的分类结果**
- 特别是在**预测知情交易概率**方面，传统规则往往优于未经迭代优化的 BVC
- BVC 的优势主要体现在系统级、聚合级别的指标计算（如 VPIN），而在逐笔判断的精度上有所折扣

> **实践建议**：在工程实现效率与微观物理真实度之间，量化分析师需要根据具体的 Alpha 频段进行权衡：
>
> - **超高频（毫秒/微秒）做市场策略**：选择 Lee-Ready，追求最高精度
> - **高频闪电崩盘预警系统**：选择 BVC + VPIN，追求实时性
> - **学术研究**：通常用 Lee-Ready 作为基准
> - **缺乏 Tick 数据的中频策略**：选择 BVC，没有别的选择

## 三、对比速查

| 特征 | Lee-Ready / Tick Rule | BVC (批量订单分类) |
|------|----------------------|-------------------|
| 底层数据颗粒度 | 逐笔成交 + 纳秒级对齐报价 | 聚合后的 K 线（时间柱或等量交易柱） |
| 微观分类哲学 | **确定性离散规则** (绝对二元：纯买/纯卖) | **连续性概率切分** (统计学比例分配) |
| 计算时间复杂度 | 极高（海量状态机比对与错位清洗） | 低（仅需聚合计算 CDF） |
| 高频碎片市场适应性 | 易受时间戳延迟、乱序到达、孤儿成交干扰 | 极强，专为现代高频架构设计 |
| 微观准确度 | 较高 | 略低但快 |
| 适用场景 | 学术研究、高精度因子验证 | 实时高频毒性监测、宏观订单流重构 |
