# Tick：最小跳价单位与逐笔数据

"Tick"在金融中有两层含义：**价格维度**的最小跳动单位，和**数据维度**的逐笔成交记录。两者本质上是同一个概念在不同场景下的投射。

> 相关：[流动性](Liquidity.md) | [行情数据 L1/L2](MarketData_L1_L2.md) | [金融数据物理层次架构](../Microstructure/DataArchitecture_Physics.md)

---

## 一、价格维度：Tick Size = 最小跳动单位

Tick Size 是交易所规定的**股票价格变动的最小单位**。

### 不同市场的 Tick Size

| 市场 | Tick Size | 例子 |
|------|----------|------|
| 美股主流股票 | **$0.01**（1 美分） | AAPL: $150.01 → $150.02 |
| 美股某些低价股 / 特定交易所 | $0.001 | — |
| 期货市场（标普 500 ES） | **0.25 点** | 每个 Tick = 0.25 × $50 = $12.50 |
| A 股 | **¥0.01** | 贵州茅台: ¥1800.01 → ¥1800.02 |

### "1~2 个 Tick 的价差"是什么意思？

**例子**：某只股票当前的买一价 (Bid) 是 $100.00，卖一价 (Ask) 是 $100.01。

$$\text{价差} = 100.01 - 100.00 = 0.01 = \text{1 个 Tick}$$

如果买一价是 $100.00，卖一价是 $100.02，则价差是 **2 个 Tick**。

### 为什么"1~2 个 Tick"被视为流动性极好的标志？

**极低的交易摩擦**：

- 你买入时支付卖一价，卖出时只能拿到买一价
- 价差 = 1 个 Tick → 买入后立刻卖出只亏 1 个 Tick（市场允许的**最低成本**）
- 价差 = 10 个 Tick → 交易的一瞬间账户直接浮亏 10 个 Tick → 对高频/超短线策略来说**无法盈利**

**订单簿非常厚**：

- 价差只有 1~2 个 Tick 说明在相邻的两个价格上有**大量挂单在争抢**
- 大资金进出时不会因为"没有对手盘"而导致价格跳空

## 二、数据维度：Tick Data = 逐笔成交数据

这就是量化研究中常说的"Tick 数据"——市场的最高分辨率数据。

### 定义

每一笔在交易所**撮合成功的交易**，或者每一个挂单 (Bid/Ask) 的变动，都会生成一条记录。

### 一条 Tick Data 的典型内容

| 字段 | 含义 | 例子 |
|------|------|------|
| **时间戳** (Timestamp) | 毫秒甚至微秒级精度 | 09:30:01.123456 |
| **成交价格** (Price) | 此时的成交价 | $150.03 |
| **成交量** (Volume) | 成交了多少股 | 200 股 |
| **买卖方标识** (Side) | 主动买入 or 主动卖出 | Buy (Aggressive) |

---

## 三、两层含义的关系

> **Tick Size 是构成 Tick Data 的基本单位。**

- 当你观察 Tick Data 时，每一笔新成交的价格**永远是 Tick Size 的整数倍**
- 如果把一段时间内所有的 Tick Data 画成折线图，这条线就是由无数个 Tick Size 的跳动**拼凑**而成的

| 概念 | 比喻 | 关注点 |
|------|------|--------|
| **Tick Size** | 尺子的最小刻度 | 价格的"分辨率"——空间维度 |
| **Tick Data** | 市场的心电图 | 每一瞬间的跳动记录——时间维度 |

---

## 四、对量化建模的影响

### 4.1 价格优化

如果在做价格优化（如用 Pyomo 构建 MINLP 模型）：

- 关心的是 **Tick Size**
- 因为它决定了模型中**决策变量的离散程度**
- 你不能把价格设为 $100.005，交易所不接受——必须限制在 Tick Size 的**整数倍**上
- 这本质上是一个**整数规划约束**

### 4.2 回测与策略实现

如果在做回测或策略验证：

- 关心的是 **Tick Data**
- 需要通过历史逐笔数据计算**滑点 (Slippage)**、**冲击成本 (Impact Cost)**
- 验证订单在历史盘口中**何时真正"成交"**
- 将离散的 Tick Data 聚合为 K 线或因子是 [高频因子挖掘](../Strategies/HighFreqFactorMining.md) 的第一步

## 五、Tick 大小分类：Small / Medium / Large Tick

单看"Tick Size = 0.01 元"这个绝对值是没有意义的——**关键是它占资产价格的比例**。同样是 0.01 元 tick，对 1800 元的茅台和 5 元的银行股，影响完全不同。

### 定义

**Tick Size 相对于资产价格的比例** 决定了资产属于哪一类 Tick：

$$\text{相对 Tick} = \frac{\text{Tick Size}}{P}$$

### Small Tick（小 Tick）

Tick size 相对价格**非常小**。

- 股价 100 元，tick size 0.01 元 → 占价格的 **0.01%**
- 典型例子：大型蓝筹股、流动性极高的股票（贵州茅台 ¥1800 量级）
- **特点**：
  - 买卖价差（bid-ask spread）通常只有 1 个 tick
  - 市场非常"连续"，价格变动细腻
  - 订单簿**拥挤**，每一档都有大量挂单争抢

### Large Tick（大 Tick）

Tick size 相对价格**较大**。

- 股价 5 元，tick size 0.01 元 → 占价格的 **0.2%**
- 典型例子：低价股、某些期货合约
- **特点**：
  - 买卖价差通常也只有 1 个 tick，但这 1 个 tick 已经代表**相当大的比例**
  - 做市商每笔交易的利润空间更大
  - 价格"跳跃感"更强
  - 订单簿相对**稀疏**，深度集中在少数档位

### Medium Tick（中 Tick）

介于两者之间，tick size 占价格比例适中。

### 三类 Tick 的对比

| 维度 | Small Tick | Medium Tick | Large Tick |
|------|-----------|------------|-----------|
| **相对 Tick (Tick/P)** | < 0.05% | 0.05% - 0.2% | > 0.2% |
| **典型资产** | 高价蓝筹（茅台） | 中价股 | 低价股、小期货合约 |
| **价差竞争** | 激烈（价差几乎为 0） | 中等 | 宽松（价差固定在 1 tick） |
| **做市商行为** | 靠**交易量**盈利 | 混合 | 每笔都有稳定利润 |
| **价格发现** | 更精细 | 中等 | 较粗糙 |
| **订单簿结构** | 拥挤的订单簿 | 中等 | 稀疏的订单簿 |
| **HFT 活跃度** | 极高（微小价差套利） | 中等 | 低（价差天花板限制） |

### 为什么这个分类重要？

在**市场微观结构 (Market Microstructure)** 研究中，Small Tick 和 Large Tick 的市场**必须分开建模**——它们的流动性机制、做市行为、价格形成过程本质不同：

1. **OFI/MLOFI 在两类市场表现不同**
   - Small Tick：价格连续变化，OFI 的线性预测能力强
   - Large Tick：价格离散跳动，需要改用分类模型（价格不动 / 上跳 1 tick / 下跳 1 tick）

2. **做市商策略差异**
   - Small Tick 市场：做市商靠**频繁抢单**在拥挤订单簿中博取头寸
   - Large Tick 市场：做市商靠**维持价差**，一笔成交就有稳定利润

3. **滑点与冲击成本的计算口径**
   - Small Tick 市场：滑点以 **bps** 或**相对值**计算，连续数学处理
   - Large Tick 市场：滑点以 **tick 数**计算，离散整数处理

4. **A 股的特殊性**
   - A 股统一使用 0.01 元 tick
   - 1-5 元区间的小盘股是**典型 Large Tick**（相对 0.2%-1%）
   - 100+ 元的蓝筹是**Small Tick**（相对 < 0.01%）
   - 同一套因子/策略在两类股票上迁移时**必须重新校准**

### 与其他概念的联系

- [Maker-Taker 模型与返佣](MakerTaker_Rebates.md)：Large Tick 市场的做市商利润主要来自价差，Small Tick 市场才更依赖返佣机制
- [平方根冲击定律](SquareRootImpactLaw.md)：在 Large Tick 市场，$\delta$ 指数会更接近 1（离散跳动 → 更接近线性冲击）
- [A 股市价申报订单类型](OrderTypes_AShare.md)：U 类"本方最优"在 Small Tick 市场是高频算法的主要工具，在 Large Tick 市场意义较弱
- [OFI / VOI](../Microstructure/OFI_VOI.md)：Large Tick 的订单簿**价格离散性**使得 MLOFI 的多档加权权重设计需要调整
