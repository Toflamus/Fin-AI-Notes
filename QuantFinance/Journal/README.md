# Journal — 研究日志

日志式随笔的归档处。记录日常零星的点子、盘中发现、数据观察、Debug 心得。

> **定位**：低摩擦的"原始素材流"。与 [Research/](../Research/) 互补——Journal 是入口，Research 是沉淀的出口。

---

## 格式约定

### 文件命名

**周粒度**，一周一个文件，用**周一的日期**命名：`YYYY-MM-DD.md`

- `2026-04-20.md` → 覆盖 2026-04-20 (周一) ~ 2026-04-26 (周日)
- `2026-04-27.md` → 覆盖 2026-04-27 ~ 2026-05-03
- ...

> 为什么是周？日粒度太碎（很多天空文件），月粒度又太臃肿（一个月几十条翻不动）。周粒度正好匹配组会/周报的复盘节奏。
> 为什么用周一日期命名？比 ISO 周数 `2026-W17.md` 可读性高，打开文件名就知道时间范围。

### 文件内结构

```markdown
# Week of 2026-04-20 (周一起)

> 本文件覆盖 2026-04-20 (周一) ~ 2026-04-26 (周日)。

## 2026-04-22
- 观察一 #tag1 #tag2
- 观察二 #tag

## 2026-04-21
- ...

## 2026-04-20
- ...
```

**约定**：
1. **倒序**：新日期在最上面，打开文件就是最新
2. **Tag**：用 `#主题` 标记，以后 `grep -r "#ofi" Journal/` 就能聚合
3. **一行一条**：一句话说不完的就该升级到 Research/

### 常用 Tag 约定

| Tag | 含义 |
|-----|------|
| `#ofi` | OFI / 订单流相关 |
| `#微观结构` | 市场微观结构观察 |
| `#因子` | 因子构建 / 评估 |
| `#数据` | 数据源 / 清洗 / 字段说明 |
| `#回测` | 回测框架 / 结果 |
| `#infra` | 环境 / 工具 / 基础设施 |
| `#组会` | 组会可用素材 |
| `#待深入` | 需要后续系统研究的点 |

## 升级规则

某条 bullet 越想越多，或同一个 tag 已积累 3-5 条 → **升级为 Research/XXX.md**。

升级后在 Journal 原 bullet 加锚点：

```markdown
- 发现 OFI 在 A 股尾盘失效 #ofi → [Research/OFI_TailFailure.md]
```

这样保留了线索的时间戳，又有了沉淀结论的入口。

---

## 不要做的事

- ❌ 纠结排版 / 措辞——Journal 的价值就是低摩擦
- ❌ 删除"看起来不重要"的条目——过期观察也是历史记录
- ❌ 把 Journal 当 Research 写——需要小标题和公式的就该去 Research/
- ❌ 每天强迫自己写——没观察就不写，比硬凑更好

---

## 复盘节奏

- **周日晚 / 周一早 5 分钟**：翻一遍上周文件，挑有价值的进组会材料或周报；顺手新建下周文件
- **月末 / 季末**：`grep -r "#tag" Journal/` 看哪些主题积累得多，考虑升级为 Research 综述
