# 高频因子挖掘：从 Tick 到 K 线的数据处理范式

## 1. 核心逻辑

高频因子计算的本质：将**极高频、高噪声、非对齐**的原始数据（Tick/Snapshot），转化为**低频、高信噪比、等间距**的特征信号（因子），并最终回归到交易目标（如 K 线收益率）。

整个过程类似于 ETL / MapReduce 范式：

```
原始 Tick 数据 → 数据变换（提取信号）→ K 线聚合（降噪压缩）→ 目标对齐（回归预测）
```

---

## 2. 三阶段流程

### 2.1 数据变换 (Data Transformation)

在原始高频时间序列上进行算术或逻辑运算，将原始的"价格"转换为具有统计意义的"信号"。

| 变换类型 | 示例 |
|----------|------|
| 中间价 | $\text{MidPrice} = (P_{bid} + P_{ask}) / 2$ |
| 订单簿不平衡度 (OBI) | $(V_{bid} - V_{ask}) / (V_{bid} + V_{ask})$ |
| 收益率 | $r_t = \ln(P_t / P_{t-1})$ |
| 成交量对数化 | $\ln(\text{Volume})$ |
| 价格冲击 | 逐笔成交对盘口的瞬时影响 |

### 2.2 K 线聚合 (Aggregation)

将变换后的高频信号压缩到离散时间窗口（1min / 5min 等），目的是**降噪**和**特征固化**。

聚合方式有两大类：

#### 维度计算 (Dimensional Calculation) — 单变量统计

对聚合窗口内的**单一序列**直接做统计推导：

```
f(Sequence_A)
```

- 常用算子：Sum、Mean、Std、Skew、Max/Min、Last
- 示例：过去 5 分钟内所有 Tick 收益率的标准差（波动率）

#### 维度匹配 (Dimensional Matching) — 多变量交叉聚合

在聚合时需要**参考另一个维度的状态**，或将两个不同频率/性质的序列匹配后再计算：

```
f(Sequence_A, Sequence_B)
```

- **条件聚合**：仅对"价格上涨时的成交量"求和 → 需要将"价格方向"与"成交量"两个维度匹配对齐
- **主从匹配**：将逐笔委托（Orders）匹配到逐笔成交（Trades），计算每笔成交背后的委托压力
- **委托-撤单匹配**：追踪 Order_ID，计算"虚假挂单率"

| 类别 | 核心逻辑 | 复杂度 | 应用场景 |
|------|----------|--------|----------|
| 维度计算 | f(A) | 较低 | 均值、波动率、极值 |
| 维度匹配 | f(A, B) | 较高 | 量价相关性、主买/主卖占比 |

> **维度匹配是维度计算的上层扩展**：维度计算处理单一信号的统计特征；维度匹配则将不同来源的数据"缝合"在一起，捕捉跨维度的逻辑关联。匹配完成后，产生的"厚特征"才会被喂给多变量模型（XGBoost 等）。

### 2.3 目标对齐与建模

将 $t$ 时刻聚合得到的因子值，与 $t + \Delta t$ 时刻的 K 线收益率（Label）进行对齐：

$$y_{t+k} = f(\text{Factor}_t)$$

**关键注意：防窥视 (Look-ahead Bias)**
- 聚合因子时不能包含聚合窗口之后的 Tick 数据
- 使用 Pandas `resample` + `offset` 可有效避免

---

## 3. 为什么必须采用这种拆解方式？

| 维度 | 说明 |
|------|------|
| 统计稳定性 | 聚合消除 Tick 级噪声，使因子分布更接近模型假设 |
| 计算效率 | 将 Tick 压缩为 K 线特征，大幅降低 ML 训练的计算压力 |
| 逻辑对齐 | 大多数策略基于 K 线或固定频率执行，确保信号与执行周期一致 |
| 防窥视 | 标准化聚合流程有效避免回测中使用"未来数据" |

---

## 4. Python 实现要点

- **时间聚合**：`pd.DataFrame.resample('5T').agg({...})`
- **维度匹配（非对称时间对齐）**：`pd.merge_asof(trades, quotes, on='timestamp')`
- **条件聚合**：`groupby` + `apply` 或布尔索引后聚合
- **防窥视**：聚合时使用 `closed='left', label='left'` + 适当 offset
